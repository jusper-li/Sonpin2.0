# AGENTS.md

## 專案工作規則

本專案的修改應以小範圍、可回溯、符合既有架構為原則。除非任務明確要求，不要進行大規模改寫或重構。

## 翻譯架構唯一規則

只使用同一套翻譯架構：

- `normalizeLang`
- `pickByLang`

所有語系判斷、文案選擇與翻譯資料讀取，都應沿用這套架構。不要混用其他翻譯方式、臨時語系判斷、硬編碼替代邏輯，或另建平行翻譯流程。

## 禁止大改

除非任務明確要求，禁止：

- 重寫整個頁面
- 重構整個資料流
- 改資料庫 schema
- 改登入權限
- 改路由架構
- 改部署設定
- 刪除既有功能
- 移除目前可用元件

## 修改原則

- 優先沿用現有元件、資料結構與工具函式。
- 修改應集中在與任務直接相關的檔案與區塊。
- 若需要超出任務範圍的大改，應先說明原因並等待確認。
- 保留目前可用功能，不因清理或簡化而移除既有行為。
